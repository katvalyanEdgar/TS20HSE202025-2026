# HW3ROWS

Репозиторий для домашнего задания по прогнозированию. Текущая версия проекта
ориентирована на трек с проверкой гипотезы:

> Локальные модели проигрывают глобальным, если временных рядов много и они
> похожи по структуре.

В репозитории реализован воспроизводимый эксперимент на подвыборке `M4 Monthly`:

- обязательные бейзлайны: `Naive`, `SeasonalNaive`, `AutoETS`, `AutoTheta`;
- кластеризация рядов: `KMeans` по вручную рассчитанным признакам;
- основная модель: `CatBoost` в трех режимах:
  - отдельная локальная модель на каждый ряд;
  - глобальная модель внутри каждого кластера;
  - одна глобальная модель на всех рядах сразу.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск

Запуск по умолчанию на `M4 Monthly` с выборкой из 120 рядов:

```bash
python run_experiment.py
```

Быстрый тест на маленькой подвыборке:

```bash
python run_experiment.py --n-series 20 --output-dir results/smoke
```

## Что сохраняется

После запуска скрипт сохраняет:

- `metrics.csv`: итоговые метрики по моделям;
- `horizon_metrics.csv`: метрики по каждому шагу горизонта;
- `cluster_assignments.csv`: номер кластера для каждого ряда;
- `cluster_feature_summary.csv`: средние признаки рядов внутри кластера;
- `silhouette_scores.csv`: качество кластеризации для разных значений `k`;
- `predictions.csv`: прогнозы всех моделей в единой таблице;
- графики в `results/plots/`.

## Логика отчета

1. Сформулировать гипотезу и объяснить, почему `M4 Monthly` подходит для ее проверки.
2. Описать выборку рядов, разбиение на обучение и тест, кластеризацию и протокол оценки.
3. Показать, что кластеры действительно различаются, опираясь на графики и сводные таблицы.
4. Сравнить локальные, кластерные глобальные и общие глобальные модели `CatBoost` с обязательными бейзлайнами.
5. Сделать вывод, где гипотеза подтверждается, а где нет.

## Примечания

- Для `CatBoost` используется масштабирование по среднему абсолютному уровню ряда, чтобы уменьшить различия по масштабу между сериями. Все метрики считаются после возврата прогноза в исходную шкалу.
- `AutoETS` и `AutoTheta` запускаются через `statsforecast`, чтобы сравнение соответствовало формулировке задания и не зависело от самодельной реализации.
